# Plan: Multi-Language Support (vi / en)

Move all static UI text into JSON translation files, support Vietnamese and English, language persisted in localStorage.

---

## Phase 0: Findings (gathered, do not re-research)

### Verified facts

- **No i18n library installed.** Choice is open.
- **~48 hardcoded strings** across exactly 3 source files:
  - `src/routes/login.tsx` — 8 strings
  - `src/routes/_authenticated/profile.tsx` — ~35 strings
  - `src/components/app-sidebar.tsx` — 5 strings
  - `src/routes/__root.tsx` — page title "TanStack Start Starter"
- **`src/lib/locale.ts`** already exports `DEFAULT_COUNTRY` / `DEFAULT_TIMEZONE` from env — extend it with `DEFAULT_LANGUAGE`.
- **AuthProvider pattern** (`src/lib/auth.tsx`) is the model to copy for an i18n provider.
- **Router context** (`src/router.tsx`) has `Wrap` prop for providers — i18n can slot in there.
- **Vite alias `~/`** maps to `src/` — use `~/locales/vi.json` etc.

### Library: `react-i18next` + `i18next`

Reasons: widely used with React 19, no vite plugin required, simple JSON format, `useTranslation()` hook matches project style, language detection via localStorage is trivial.

## Phase 1: Install + Bootstrap i18n

### 1a. Install packages

```bash
npm install i18next react-i18next
```

### 1b. Add env var to `.env.local`

```
VITE_DEFAULT_LOCALE=vi-VN
```

### 1c. Extend `src/lib/locale.ts`

Add:

```ts
export const DEFAULT_LANGUAGE = import.meta.env.VITE_DEFAULT_LOCALE ?? 'vi-VN'
```

### 1d. Create `src/lib/i18n.ts`

```ts
import i18n from 'i18next'
import { initReactI18next } from 'react-i18next'
import { DEFAULT_LANGUAGE } from '~/lib/locale'
import vi from '~/locales/vi.json'
import en from '~/locales/en.json'

const LANG_KEY = 'giaoly_lang'

const savedLang =
  typeof localStorage !== 'undefined'
    ? (localStorage.getItem(LANG_KEY) ?? DEFAULT_LANGUAGE)
    : DEFAULT_LANGUAGE

i18n.use(initReactI18next).init({
  resources: {
    vi: { translation: vi },
    en: { translation: en },
  },
  lng: savedLang,
  fallbackLng: 'vi',
  interpolation: { escapeValue: false },
})

export { LANG_KEY }
export default i18n
```

> Note: bundled JSON (no HTTP fetch) — works offline and in SSR without async loading.

### 1e. Register provider in `src/routes/__root.tsx`

Import and add `I18nextProvider` wrapping inside `RootDocument`:

```tsx
import { I18nextProvider } from 'react-i18next'
import i18n from '~/lib/i18n'

// In RootDocument:
;<AuthProvider>
  <I18nextProvider i18n={i18n}>
    <TooltipProvider>{children}</TooltipProvider>
  </I18nextProvider>
</AuthProvider>
```

> Import `~/lib/i18n` (side-effect: calls `init`) before the provider so it's always initialized.

### 1f. Verification

- `npx tsc --noEmit` passes
- `import i18n from '~/lib/i18n'; console.log(i18n.language)` prints `vi` in browser console

## Phase 3: Wire `useTranslation` into existing files

Pattern to follow (copy from login.tsx as template):

```tsx
import { useTranslation } from 'react-i18next'

function MyComponent() {
  const { t } = useTranslation()
  return <Label>{t('auth.loginId')}</Label>
}
```

Validator error messages also use `t`:

```ts
onBlur: ({ value }) => {
  const r = z.string().min(1).safeParse(value)
  return r.success ? undefined : t('auth.loginId.required')
}
```

> `t` from `useTranslation()` is a stable reference — safe inside validator callbacks.

## Phase 4: Language Switcher

### 4a. Language change util in `src/lib/i18n.ts`

```ts
export function setLanguage(lang: string) {
  localStorage.setItem(LANG_KEY, lang)
  i18n.changeLanguage(lang)
}
```

### 4b. Add switcher to `NavUser` dropdown in `src/components/app-sidebar.tsx`

Below the "Hồ sơ" item, above the logout separator:

```tsx
import { setLanguage } from '~/lib/i18n'

// Inside NavUser dropdown:
<DropdownMenuSeparator />
<DropdownMenuItem onClick={() => setLanguage('vi')}>
  🇻🇳 Tiếng Việt
</DropdownMenuItem>
<DropdownMenuItem onClick={() => setLanguage('en')}>
  🇬🇧 English
</DropdownMenuItem>
```

> Language change is instant — `react-i18next` re-renders all `useTranslation` consumers automatically.

### Verification

- Click "English" → all UI strings switch to English
- Reload → English persists (from localStorage)
- Click "Tiếng Việt" → switches back

### Anti-patterns to avoid

- Do NOT use `i18n.t()` (module-level) — always `useTranslation()` inside components for reactivity
- Do NOT wrap `navItems` array outside a component — `t()` requires component render context
- Do NOT add SSR language detection — keep it localStorage-only to avoid hydration mismatch
- Do NOT split namespaces — single `translation` namespace is fine for this app size
